4. `Tomcat`集群会带来什么问题？

+ `Session`登录信息存储及读取问题  
+ 服务器定时任务并发问题  
+ ....  

###### 解决方案

+ 采用`nginx ip hash policy `   
 优点：可以不改变现有技术架构，直接实现横向扩展（省事）   
 缺点：  
 1.导致服务器请求（负载）不平均（完全依赖`ip hash`的结果）  
 2.在`ip`变化的环境下无法服务
